Why Stream Healer?

Broken JSON is a "solved" problem that still crashes production apps. Inspired by OpenRouter's research showing 80%+ defect reduction, we bring that same reliability to your local and edge streams.

data_object

Auto-balancing Braces

Automatically maintains a stack of open braces and brackets, closing them intelligently to ensure valid JSON structure at every chunk.

code_off

String Termination

Smartly detects cut-off strings, handles partial escape sequences (like `\n` or `\"`), and closes quotes before validation.

bolt

Zero Dependencies

A lightweight, focused library with no external bloat. Perfect for edge functions, serverless environments, and browser usage.

javascript

TypeScript Native

Written in TypeScript with full type definitions included. Enjoy autocomplete and type safety right out of the box.

tune

Flexible Modes

Choose between strict validation (`json_schema`) or flexible structure (`json_object`). Activated automatically via your request's `response_format`.

speed

High Performance

Optimized string manipulation ensuring minimal overhead, even when processing high-throughput token streams.

Zero-code Integration

Don't rewrite your codebase. Just spin up the proxy and point your OpenAI client to it. We handle the stream healing transparently.

1

Start the Proxy

Run bun run src/proxy.ts (or use the binary).

2

Point your Client

Set baseURL to http://localhost:1143/v1.

3

Request JSON

Add response_format: { type: 'json_schema' } (or json_object).

client.ts 
const client = new OpenAI({
  baseURL: "http://localhost:1143/v1",
  apiKey: "ollama"
});

const response = await client.chat.completions.create({
  model: "llama3",
  messages: [...],
  stream: true,
  // âœ… Activates Healer
  response_format: { 
    type: "json_object" 
  }
});

for await (const chunk of response) {
  // Guaranteed valid partial JSON!
  process.stdout.write(chunk.choices[0]?.delta?.content);
}
